Detailed Description

Road object containing a basic line string and meta data associated with the underlying real world road. A road can be one or two way and is used in spatial indexing. A road should not be confused with an edge used in the graph topology.

Construct a road. Most fields are copied to the new object. The geometric string is used to build the underlying line string. This string should be a valid WKB representation of a line string, otherwise the road will be set to invalid. This also constructs a bounds for this road so that it can be spatially indexed (e.g. by a quad tree).
